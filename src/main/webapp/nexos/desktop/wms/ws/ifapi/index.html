Nexos 표준 인터페이스 - IFAPI
Nexos 표준 인터페이스 중 RESTful 방식으로 인터페이스할 경우 호출 방법 설명
서비스 Url
기본 Url

서비스 호출 방식

고객사 송신 --> WMS 수신: Post

고객사 수신 <-- WMS 송신: Get


서비스 호출 인증

HTTP Basic Authentication 방식 사용 함

WMS에서 인증 사용시 서비스 호출 Http Header에 인증 정보 포함하여 호출하여야만 처리 가능

사용자ID/PWD는 WMS에서 부여 받아서 사용


Authorization: Basic <credentials>

데이터 구조 설명
항목 데이터타입 필수여부 설명 비고
buCd String ● 사업부코드 해당 데이터를 처리할 사업부
ediDiv String ● 수신구분 정의된 수신구분, 상품마스터: "RCM10", 공급처마스터: "RCM20", ...
defineNo String ● 수신정의번호 해당 데이터를 처리할 수신정의번호, 기본값: "IFAPI"

서비스 호출 데이터 샘플
구분 메서드명 샘플 설명
공통수신 receive.do Request Parameter로 호출:
  Url: http://.../ifapi/predefined/receive.do
  Http Method: Post
  Authorization: Basic <credentials>
  Parameter Key: payload
  Parameter Value: { ... }

Request Body로 호출:
  Url: http://.../ifapi/predefined/receive.do
  Http Method: Post
  Authorization: Basic <credentials>
  Body: { ... }
